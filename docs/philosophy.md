# Artli の前提思想と実装判断基準

このドキュメントは Artli プロジェクトが大切にしている前提思想と、UI / 機能実装の判断基準をまとめたものです。新しい機能を追加する際や迷ったときの判断軸として参照してください。

## 1. Artli の前提思想

Artli は、AI とイラストレーターの共存を目指す、**作家主体の公開型「学習許諾・権利流通市場」** です。

「AI にイラストを使わせないか、使わせるか」という二者択一ではなく、**作家が自分の作品について公開方針を示し、開発者はそれを踏まえて正規ルートで取得する**という世界を前提としています。

### 3つの両立目標

Artli は以下の 3 点を同時に満たすことを目指します。

1. **作家の管理権** — 作家自身が、自分の作品について誰にどのような条件で使わせるかを決められる
2. **開発者の正規取得ルート** — 開発者（AI 事業者・制作会社など）が、怪しいルートではなく明示された条件のもとで作品を取得できる
3. **条件・取得履歴の証跡** — いつ、誰が、どの作品を、どの条件で取得したかが記録として残る

### UI / 体験の方向性

Artli は **商業色の強い EC サイト寄せではない**プロダクトです。体験としては「次世代 pixiv」寄りの、投稿型 SNS に近い見え方を目指します。

- 作品が主役、作家が前面
- 条件（ライセンス）は「作家の公開方針」として自然に見える
- developer 向けの調達効率（価格の並び、取得可否でのソートなど）は主役にならない
- 業務 UI / 管理画面のような見せ方はしない

## 2. 実装判断の最重要基準

機能追加・UI 変更で迷ったときは、以下の基準で判断します。

### 最重要: 「完成形としてイラストレーター優勢に見えるか」

Artli は作家主体のプロダクトです。画面を完成形で眺めたとき、**イラストレーター側が主役に見えるか**が最も重要な判断基準です。

### 具体的な重視事項

- **開発者向けの取得・調達画面のように見せない**
  - 価格比較・取得可否ソート・一括取得導線などは主役にしない
- **まず作品や作家に出会う場として見せる**
  - 一覧やトップの第一印象は「作品」「作家」であるべき
- **ライセンス条件は作家が示したルール / 公開スタンスとして見せる**
  - 条件を「商品スペック」のようには見せない
- **価格、取得可否、即時取得可能性を前面に出しすぎない**
  - 必要な場所には出るが、主役カードの第一層には出さない
- **UI はできるだけ軽く、投稿型 SNS らしい見え方を保つ**
  - 左サイドバーに多数のフィルタ、比較表、業務画面のような密度は避ける

---

## 3. 判断に迷ったときの優先順位

新機能や UI 変更の議論で意見が割れた場合は、以下の順に優先します。

1. 作品と作家が主役に見えるか
2. 完成形としてイラストレーター優勢に見えるか
3. 既存の public pages / Prisma schema / License モデルを壊していないか
4. MVP として小さく閉じられているか（過剰実装になっていないか）
5. これまでの導線（作家投稿 → 公開 → 開発者取得）と自然につながっているか

この順序は個別の Prompt（実装単位）の設計でも踏襲されます。
